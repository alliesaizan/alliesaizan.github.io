What makes a comic character successful?

Posted on December 21, 2018

In a previous post, I found that in both Marvel and DC comics, an increasing percentage of new characters are female each year. Further, Marvel has done a better job than DC post-2000 in releasing LGBT characters. However, I wasn’t able to ascertain whether these characteristics impact a character’s popularity.

The outcome that answers this question is “number of appearances since character’s release.” Appearance is a measure of popularity; the higher a character’s number of appearances, the more popular s/he has been throughout its tenure.

To predict number of appearances, I will first use a simple regression tree analysis.

I’ll use the following predictors: publisher, gender, sexual orientation, whether hero/villian identity is a secret, alignment (good or evil), age, and whether the character ever died. The table below shows a quick summary of the predictors.

The data with numeric columns contains 23,272 rows and 10 variables. The first step I need to take is splitting the data into train and test sets. I’ll choose a 70-30 split between train and test, respectively.

Regression Tree Modeling

I train a regression tree model using the rpart package.

The most important variable is the variable used to make the first split. The tree above shows that character age is most important in predicting appearances. The algorithm finds certain age splits to be useful at different levels. Other important factors, in decreasing order, include character alignment and publisher.

To minimize overfitting, I need to pick a complexity parameter that minimizes the tree’s error. The complexity parameter controls the size of the tree (the number of splits) such that the tree is at its optimal size. Looking at the CP plot below, where we plot the different values of the complexity parameter against the cross validation error, a CP value of 0.01 appears to generate the smallest cross validation error. Inspecting the the fit summary shows that the tree is already set at CP parameter of 0.01.

The average number of appearances skews low. The boxplot above shows that most characters have few appearances (as might be expected by publishers introducing one-hit wonders for particular storylines, for instance). The lack of options for appearances suggests that the model may have overfitted the training set; that is, the rules used to generate the regression tree on the training data may not generalize well to new data. Random forest modeling presents an alternative to regression tree modeling that is more robust to overfitting.

Random Forests

Random forest modeling builds a great many decision trees using samples of the data, aggregates those trees (hence the “forest” moniker), and averages the prediction for the response variable for continuous outcomes. At each node of a given tree, the algorithm chooses a random sample of predictors to make the next split. Random Forest modeling is robust to out-of-sample (test) data and can provide interesting insights into variable importance.

Unfortunately, R’s random forest function (randomForest) only accepts data with no missing values. But I am not in luck; the visualization below shows that the comics data is missing in many places. The Secret_Identity variable suffers the most from missingness.

I deal with missings in the following way: - For continuous variables, I replace missings with the median value. - For binary variables, I replace missing values with zero.

Now, I can run the random forest algorithm on my new, imputed data set.

The random forest package produces two measures of variable importance: percent increasing mean squared error, and increase in node purity.

Interestingly, the random forest model shows that whether the character ever died is the most important variable to the number of comic appearances. Characters often die in big, cross-over events, and after they are usually rebooted in a new series or line of comics. More popular characters are more likely to appear in “big event” storylines or be in need of rebooting, and so it makes sense that the ever_died variable acts as a proxy for a character’s popularity. Other important factors include whether the character identifies as LGBT, whether s/he has a good alignment, and the character’s comic book publisher.

Summary

Now, I’ll discuss the caveats of my analysis and ideas for future research.

I’m not super happy with the way I dealt with missings. Imputing with the median is a rather crude way of filling in missing data, and in future analyses, I’d like to implement a more sophisticated imputation method.

I also think my prediction would be more robust if I had more features. In the process of searching for data, I also came across an Avengers database. Merging the Marvel subset of this comics data with the Avengers data might provide interesting insights. I wasn’t able to match the two datasets together easily because of differences in the merging columns. One downside of incorporating that additional data is that while it feels to me like every third Marvel character is associated with the Avengers in some way these days, I probably would have a more severe problem with missing data if I incorporated Avengers data.
